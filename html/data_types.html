• A data type defines:
    • a collection of data values
    • a set of predefined operations on those values
  • Importance of data types in determining ease of data manipulation in a program
  • Evolution of data typing concepts over 60 years
    • Earliest languages: basic language-supported data structures modeled problem space data structures
      • Example: pre-90 Fortrans
    • COBOL: first step away from basic data structures; accuracy specification for decimal, structured data type for records
    • PL/I: accuracy specification for integer and floating-point types, many data types to support large range of applications
    • ALGOL 68: few basic types and flexible structure-defining operators for designing data structures
    • User-defined types:
      • Improved readability through use of meaningful names
      • Type checking of special category of variables
      • Aid modifiability: change type of category of variables by changing type definition statement only
  • Abstract data types:
    • Idea: interface visible to user separated from representation and operations, which are hidden
    • Supported by most programming languages since mid-1980s
